# RDQAS(R Data Quality Assessment System)

R 기반 사용자 친화형 데이터 품질 검사 시스템

## 프로젝트 개요

본 프로젝트는 R 기반의 사용자 친화형 데이터 품질 검사 시스템을 개발하는 것이 목적입니다. 사용자가 CSV 또는 XLSX 데이터를 업로드하면 데이터의 기본 구조를 확인하고, 결측치, 이상치, 중복 행, 자료형 및 형식 문제 등 데이터 품질 문제를 자동으로 진단합니다.

또한 사용자가 선택한 분석 목적에 대해 현재 데이터가 필요한 변수 구조와 표본 조건을 갖추고 있는지 평가합니다. 분석 목적별 적합성 평가는 데이터 품질 점수와 별도로 제공되며, 사용자는 데이터 자체의 품질 문제와 특정 분석 목적에 대한 한계를 구분하여 확인할 수 있습니다.

RDQAS는 실제 통계분석이나 머신러닝 모델을 직접 수행하는 시스템이 아니라, 분석 전 단계에서 데이터 품질과 분석 목적별 적합성을 사전 점검하기 위한 보조 시스템입니다. 이를 통해 사용자는 데이터의 현재 상태를 직관적으로 이해하고, 분석 전 필요한 전처리 방향과 적절한 분석 목적을 판단할 수 있습니다.

## 주요 제공 기능

### 1. 데이터 입력 기능

사용자가 CSV 또는 XLSX 파일을 업로드하면 이를 R의 데이터프레임 형태로 불러옵니다. 업로드된 데이터에 대해 행 수, 열 수, 변수명, 변수 유형, 데이터 미리보기를 제공합니다.

### 2. 변수 유형 자동 판정 기능

업로드된 데이터의 각 변수를 분석 관점에서 자동 판정합니다. 변수 유형은 수치형 변수, 범주형 변수, 숫자형 범주 변수, 날짜형 변수, ID성 변수, 제외 변수 등으로 구분됩니다.

변수 유형 판정 결과는 데이터 품질 진단과 분석 목적별 적합성 평가에서 공통 기준으로 활용됩니다.

### 3. 데이터 품질 진단 기능

입력된 데이터에 대해 다음 항목을 점검합니다.

* 결측치
* 이상치
* 중복 행
* 자료형 및 형식 문제
* 변수 구조 문제
* 범주형 변수 품질
* 데이터 기본 구조 문제

진단 결과는 데이터 품질 점수, 항목별 점수, 주요 경고, 컬럼별 품질 상세 정보로 제공됩니다.

### 4. 분석 목적별 적합성 평가 기능

사용자가 선택한 분석 목적에 대해 현재 데이터가 필요한 변수 구조와 표본 조건을 갖추고 있는지 평가합니다.

지원하는 분석 목적은 다음과 같습니다.

* 기초 통계 분석
* 범주형 변수 빈도 분석
* 통계적 검정 / 가설검정
* 상관관계 분석
* 회귀분석
* 분류분석
* 군집분석
* 시계열분석
* 차원 축소
* 생존분석

적합성 평가는 분석 수행에 반드시 필요한 필수 기준과, 분석 결과의 안정성을 판단하는 권장 기준으로 구분됩니다. 최종 결과는 적합, 부분 적합, 부적합으로 제공됩니다.

### 5. 품질 시각화 기능

Step 3 품질 시각화 화면에서 데이터 품질 문제가 두드러지는 변수를 그래프로 확인할 수 있습니다.

* 결측률 상위 변수 막대그래프
* 이상치 비율 상위 수치형 변수 박스플롯
* 결측치가 없는 경우 안내 문구 출력
* 이상치가 탐지된 수치형 변수가 없는 경우 안내 문구 출력

### 6. HTML 보고서 생성 기능

진단 결과와 적합성 평가 결과를 HTML 보고서로 저장할 수 있습니다. 보고서에는 다음 내용이 포함됩니다.

* 요약 정보
* 데이터 품질 점수
* 분석 적합성 점수
* 선택 변수 정보
* 데이터 품질 평가 결과
* 분석 적합성 평가 결과
* 컬럼별 품질 상세
* 품질 시각화
* 주요 발견사항 및 권고사항

생성된 보고서는 사용자가 직접 다운로드할 수 있으며, 프로젝트 내부의 `reports/` 폴더에도 저장할 수 있습니다.

## 개발 규칙

### 1. 파일명 규칙

* 파일명에는 공백을 사용하지 않습니다.
* 단어 구분은 언더바 `_`를 사용합니다.
* R 코드 파일은 `.R` 확장자를 사용합니다.
* 주요 R 파일은 실행 순서와 역할이 드러나도록 `번호_기능명.R` 형식으로 작성합니다.

예시는 다음과 같습니다.

```text
01_packages.R
02_report.R
03_ui.R
04_server.R
```

### 2. 경로 작성 규칙

코드 안에서는 개인 컴퓨터에만 적용되는 절대경로를 사용하지 않습니다.

사용하지 않는 예시는 다음과 같습니다.

```r
source("C:/Users/Owner/Desktop/RDQAS/R/01_packages.R")
```

사용하는 예시는 다음과 같습니다.

```r
source("R/01_packages.R")
```

상대경로를 사용해야 다른 사용자가 저장소를 clone한 뒤에도 동일하게 실행할 수 있습니다.

### 3. main 브랜치 관리 규칙

* `main` 브랜치에는 실제 실행에 필요한 최종 파일만 유지합니다.
* 수정이 필요한 경우 별도의 작업 브랜치에서 수정한 뒤 Pull Request를 생성합니다.
* 작업 브랜치에서 기능 테스트와 실행 오류 확인을 마친 뒤 병합합니다.
* `final1`, `final2`, `candidate`, `test`, `backup` 등 임시 파일은 `main` 브랜치에 남기지 않습니다.

### 4. 브랜치 및 커밋 규칙

본 프로젝트는 팀원별 담당 역할이 나누어져 있으므로, 브랜치는 개인 또는 담당 영역 기준으로 생성할 수 있습니다.

브랜치 이름 예시는 다음과 같습니다.

```text
cj_pm
jw_quality
gr_suitability
hn_report
```

커밋 메시지는 무엇을 수정하거나 추가했는지 명확하게 작성합니다.

예시는 다음과 같습니다.

```text
데이터 품질 진단 함수 분리
분석 적합성 평가 기준 보완
품질 시각화 기능 추가
HTML 보고서 저장 오류 수정
README 파일 구조 및 사용 방법 수정
```

커밋 메시지 작성 시 다음 원칙을 따릅니다.

* 변경 내용을 한눈에 알 수 있도록 구체적으로 작성합니다.
* 가능하면 한 커밋에는 하나의 작업 단위만 포함합니다.
* 기능이 정상 실행되는지 확인한 뒤 커밋합니다.

## RDQAS 사용 방법

### 1. 저장소 클론

먼저 GitHub 저장소를 로컬 환경에 clone합니다.

```bash
git clone https://github.com/Deity219/RDQAS
cd RDQAS
```

### 2. RStudio에서 프로젝트 열기

RStudio에서 `RDQAS.Rproj` 파일을 열거나, RStudio의 `File > Open Project` 메뉴를 통해 RDQAS 프로젝트를 엽니다.

이때 작업 디렉토리(Working Directory)는 반드시 `RDQAS` 프로젝트의 최상위 폴더여야 합니다.

```r
getwd()
```

위 명령어를 실행했을 때 경로가 다음과 같이 `RDQAS` 폴더를 가리키면 정상입니다.

```text
.../RDQAS
```

### 3. 필요한 R 패키지 설치

처음 실행하는 사용자는 아래 패키지를 설치해야 합니다.

```r
install.packages(c(
  "shiny",
  "shinyBS",
  "ggplot2",
  "dplyr",
  "tidyr",
  "knitr",
  "rmarkdown",
  "readxl",
  "readr",
  "DT",
  "shinydashboard",
  "shinyjs",
  "plotly",
  "htmltools",
  "base64enc",
  "scales",
  "lubridate",
  "survival"
))
```

`R/01_packages.R`에서도 필수 패키지 설치 여부를 확인하며, 누락된 패키지가 있을 경우 설치가 필요한 패키지 목록과 설치 명령어를 안내합니다.

### 4. 앱 실행

프로젝트 최상위 폴더에서 다음 명령어를 실행합니다.

```r
shiny::runApp()
```

또는 RStudio에서 `app.R` 파일을 열고 `Run App` 버튼을 클릭하여 실행할 수 있습니다.

### 5. 기본 사용 흐름

RDQAS의 사용자 흐름은 다음 네 단계로 구성됩니다.

#### Step 1. 데이터 업로드 및 분석 목적 선택

1. CSV 또는 XLSX 파일을 업로드합니다.
2. 분석 목적을 선택합니다.
3. 분석 목적에 따라 필요한 변수를 선택합니다.
4. 데이터 미리보기를 통해 업로드 결과를 확인합니다.

#### Step 2. 진단 결과 확인

1. 데이터 품질 점수를 확인합니다.
2. 주요 경고 문구를 확인합니다.
3. 분석 목적별 적합성 평가 결과를 확인합니다.
4. 적합성 점수, 판정 상태, 권고사항을 확인합니다.

#### Step 3. 품질 시각화 확인

1. 결측률이 높은 변수를 그래프로 확인합니다.
2. 이상치 비율이 높은 수치형 변수를 박스플롯으로 확인합니다.
3. 결측치 또는 이상치가 없는 경우 안내 문구를 확인합니다.

#### Step 4. HTML 보고서 저장

1. HTML 보고서를 사용자의 PC로 다운로드합니다.
2. 필요한 경우 프로젝트 내부 `reports/` 폴더에 보고서를 저장합니다.
3. 생성된 보고서에서 품질 진단 결과, 적합성 평가 결과, 품질 시각화, 권고사항을 종합적으로 확인합니다.

## 주의사항

* RDQAS는 분석 전 데이터 품질과 분석 목적별 적합성을 점검하는 시스템입니다.
* 실제 회귀모형 적합, 분류모델 학습, 군집 알고리즘 실행, 생존분석 모형 추정 등 실제 분석 실행 기능은 포함하지 않습니다.
* 데이터 품질 점수와 분석 적합성 점수는 서로 다른 개념입니다.
* 데이터 품질 점수가 높더라도 특정 분석 목적에 필요한 변수 구조가 부족하면 해당 분석에는 부적합할 수 있습니다.
* 반대로 데이터 품질 문제가 일부 존재하더라도 특정 분석 목적의 기본 구조를 충족하면 분석 적합성은 높게 평가될 수 있습니다.
