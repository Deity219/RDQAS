# RDQAS(R Data Quality Assessment System)
R 기반 사용자 친화형 데이터 품질 검사 시스템

## 프로젝트 개요
본 프로젝트는 R 기반의 사용자 친화형 데이터 품질 검사 시스템을 개발하는 것이 목적입니다.
사용자가 입력한 데이터에 대해 기초 통계 요약/시각화/품질 진단/데이터와 사용자 목적 간 적합성 평가를 자동으로 수행하고 그 결과를 보고서 형태로 제공합니다. 최근 비전공자와 초보자도 데이터를 직접 활용하는 경우가 늘고 있지만, 실제 데이터에는 결측치/이상치/중복값/변수 형식 오류 등 다양한 품질 문제가 존재해 분석 신뢰성을 떨어뜨릴 수 있습니다. 이에 본 시스템은 초보자도 데이터의 상태와 활용 가능성을 쉽게 이해하고 올바른 분석 방향을 설정할 수 있도록 지원하며, 데이터 전처리와 탐색적 분석에 대한 학습 보조 도구로도 활용될 수 있습니다. 또한 반복적인 품질 점검과 기초 시각화 작업을 자동화함으로써 분석 오류를 줄이고 전체적인 분석 효율성을 높이는 효과를 기대할 수 있습니다.

## 주요 제공 기능
### 1. 데이터 입력 기능
사용자가 CSV 또는 XLSX 파일을 업로드하면 이를 자동으로 불러옵니다.

### 2. 데이터 품질 검사 기능
입력된 데이터에 대해 결측치/이상치/중복값/자료형 오류/범주형 변수의 불균형/변수별 분포 특성 등을 점검합니다.

### 3. 기초 통계 및 시각화 기능
수치형 변수에 대한 평균, 중앙값, 분산, 사분위수 등을 제공하고, 히스토그램, 박스플롯, 산점도, 막대그래프 등 기초 시각화를 자동 생성합니다.

### 4. 데이터 품질 점수 산출 기능
결측치 비율, 중복값 비율, 이상치 비율, 형식 일관성 등을 종합하여 데이터 품질 점수를 계산합니다.

### 5. 사용 목적 기반 데이터 적합성 평가 기능
사용자가 입력한 데이터 활용 목적을 바탕으로 현재 데이터가 해당 목적에 적합한지 평가하고, 그 결과를 점수 또는 안내 문장 형태로 제공합니다.

### 6. 자동 보고서 생성 기능
점검 결과, 시각화 자료/품질 점수 등을 종합하여 HTML 또는 PDF 형식의 보고서를 자동 생성합니다.

## 역할 분담
- **이찬재: PM / 기획 총괄**  
 전체 일정 관리, 요구사항 정리, 기능 우선순위 조정, 팀원 간 진행 상황 점검, 최종 산출물 검토 및 문서 작성 지원

- **유지웅: 데이터 입력 및 품질 진단 담당**  
데이터 입력 기능 구현, 변수 구조 인식, 결측치·이상치·중복값·자료형 오류 점검 기능 구현

- **이규림: 품질 점수 및 적합성 평가 담당**  
데이터 품질 점수 산정 기준 설계, 품질 점수 계산 로직 구현, 사용자 목적 기반 데이터 적합성 평가 기준 정의 및 구현

- **최한나: 시각화 및 보고서 담당**  
기초 통계 요약, 자동 시각화 구현, 결과 화면 구성, HTML/PDF 보고서 생성, 최종 테스트 및 발표 자료 정리

## 파일 구조

RDQAS는 기능별로 파일을 분리하여 관리합니다.

```text
RDQAS/
│
├─ app.R
│
├─ R/
│  ├─ 01_packages.R
│  ├─ 02_report.R
│  ├─ 03_ui.R
│  ├─ 04_server.R
│  │
│  ├─ quality/
│  │  └─ 향후 추가 예정
│  │
│  └─ suitability/
│     └─ 향후 추가 예정
│
├─ reports/
│
└─ README.md
```

### 주요 파일 설명

| 파일명 | 역할 |
|---|---|
| `app.R` | 전체 앱 실행 파일입니다. 필요한 R 파일을 불러오고 Shiny 앱을 실행합니다. |
| `R/01_packages.R` | 앱 실행에 필요한 R 패키지를 불러옵니다. |
| `R/02_report.R` | HTML 보고서 생성 함수를 정의합니다. |
| `R/03_ui.R` | 사용자가 보는 화면 구성을 정의합니다. |
| `R/04_server.R` | 파일 업로드, 진단 실행, 결과 출력 등 앱의 동작을 처리합니다. |
| `reports/` | 생성된 보고서 파일을 저장하는 폴더입니다. |

데이터 품질 진단 관련 파일은 `R/quality/` 폴더에 저장합니다.


## 개발 규칙

### 1. 파일명 규칙

- 파일명에는 공백을 사용하지 않습니다.
- 단어 구분은 언더바 `_`를 사용합니다.
- R 코드 파일은 `.R` 확장자를 사용합니다.
- 주요 R 파일은 실행 순서와 역할이 드러나도록 `번호_기능명.R` 형식으로 작성합니다.

예시는 다음과 같습니다.

```text
01_packages.R
02_report.R
03_ui.R
04_server.R
```

품질 진단 및 적합성 평가 기능을 세분화할 경우에도 같은 형식을 사용합니다.

```text
02_check_missing.R
04_check_outliers.R
08_assess_data_quality.R
13_evaluate_suitability.R
```

### 2. 경로 작성 규칙

코드 안에서는 개인 컴퓨터에만 적용되는 절대경로를 사용하지 않습니다.

사용하지 않는 예시는 다음과 같습니다.

```r
source("C:/Users/Owner/Desktop/RDQAS/R/01_packages.R")
```

사용하는 예시는 다음과 같습니다.

```r
source("R/01_packages.R")
```

상대경로를 사용해야 다른 사용자가 저장소를 clone한 뒤에도 동일하게 실행할 수 있습니다.

### 3. main 브랜치 관리 규칙

- `main` 브랜치에는 실제 실행에 필요한 최종 파일만 유지합니다.
- 수정이 필요한 경우 `main` 브랜치에서 직접 작업하지 않고, 별도의 작업 브랜치를 생성합니다.
- 작업 브랜치에서 기능 테스트와 실행 오류 확인을 마친 뒤 Pull Request를 생성합니다.
- PR 검토 후 문제가 없을 때만 `main` 브랜치에 병합합니다.
- `final1`, `final2`, `candidate`, `test`, `backup` 등 임시 파일은 `main` 브랜치에 남기지 않습니다.

### 4. 브랜치 및 커밋 규칙

본 프로젝트는 팀원별 담당 역할이 나누어져 있으므로, 브랜치는 개인 또는 담당 영역 기준으로 생성합니다.

브랜치 이름은 다음과 같이 간단하게 작성합니다.

```text
이름_담당영역
```

예시는 다음과 같습니다.

```
cj_pm
jw_quality
gr_score
hn_report
```

`main` 브랜치에서는 직접 작업하지 않고, 각자 담당 브랜치에서 수정 및 테스트를 진행합니다. 작업이 완료되면 Pull Request를 생성하고, 실행 오류가 없는지 확인한 뒤 `main` 브랜치에 병합합니다.

커밋 메시지는 **무엇을 수정하거나 추가했는지 명확하게 작성하는 것**을 우선합니다.

예시는 다음과 같습니다.

```text
CSV 파일 업로드 기능 추가
결측치 검사 오류 수정
품질 점수 계산 기준 수정
보고서 다운로드 기능 수정
UI 결과 화면 수정
README 사용 방법 추가
서버 로직 파일 분리
```

커밋 메시지 작성 시 다음 원칙을 따릅니다.

- 변경 내용을 한눈에 알 수 있도록 구체적으로 작성합니다.
- 가능하면 한 커밋에는 하나의 작업 단위만 포함합니다.
- 기능이 정상 실행되는지 확인한 뒤 커밋합니다.

## RDQAS 사용 방법

### 1. 저장소 클론

먼저 GitHub 저장소를 로컬 환경에 clone합니다.

```bash
git clone https://github.com/Deity219/RDQAS
cd RDQAS
```

### 2. RStudio에서 프로젝트 열기

RStudio에서 `RDQAS` 폴더를 열거나, 프로젝트 파일이 있는 경우 `RDQAS.Rproj` 파일을 실행합니다.

이때 작업 디렉토리(Working Directory)는 반드시 `RDQAS` 프로젝트의 최상위 폴더여야 합니다.

```r
getwd()
```

위 명령어를 실행했을 때 경로가 다음과 같이 `RDQAS` 폴더를 가리키면 정상입니다.

```text
.../RDQAS
```

### 3. 필요한 R 패키지 설치

처음 실행하는 사용자는 아래 패키지를 설치해야 합니다.

```r
install.packages(c(
  "shiny",
  "shinyBS",
  "ggplot2",
  "dplyr",
  "tidyr",
  "knitr",
  "rmarkdown",
  "readxl"
))
```

### 4. 앱 실행

프로젝트 최상위 폴더에서 다음 명령어를 실행합니다.

```r
shiny::runApp()
```

또는 RStudio에서 `app.R` 파일을 열고 `Run App` 버튼을 클릭해 실행할 수 있습니다.

### 5. 기본 사용 흐름

1. CSV 또는 XLSX 파일을 업로드합니다.
2. 데이터 분석 목적을 선택합니다.
3. 필요한 경우 분석에 사용할 변수를 선택합니다.
4. `품질 진단 시작` 버튼을 클릭합니다.
5. 데이터 품질 점수, 주요 경고, 적합성 평가 결과를 확인합니다.
6. 필요한 경우 HTML 보고서를 다운로드합니다.
